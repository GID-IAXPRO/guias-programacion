Tema 5. Polimorfismo
1. Brevemente, ¿qué es el "polimorfismo" y para qué sirve en programación orientada a objetos? ¿qué es la "sobreescritura" de métodos?
🔹 Polimorfismo

👉 Mismo tipo general → distintos comportamientos
👉 Un Soldado puede ser Zapador o Artillero

✔ Sirve para:

código genérico
más flexibilidad
reutilización
🔹 Sobreescritura (override)

👉 Subclase redefine método del padre
👉 MISMA firma, distinto comportamiento

💡 Idea: cada objeto responde a su forma

🔹 Ligadura dinámica

👉 El método se decide en tiempo de ejecución

🔹 Relación

👉 Permite el polimorfismo ✔

🔹 Lenguajes
Java → automático ✔

👉 Se ejecuta el método del objeto real, no del tipo de referencia

3. Pon un ejemplo sencillo en Java, de un Soldado...
🔹 Idea

👉 Array de Soldado con distintos tipos
👉 Cada uno ejecuta su saludar()

class Soldado {
    public void saludar() {
        System.out.println("Soy soldado");
    }
}

class Zapador extends Soldado {
    public void saludar() {
        System.out.println("Soy zapador");
    }
}

class Artillero extends Soldado {
    public void saludar() {
        System.out.println("Soy artillero");
    }
}

Soldado[] soldados = {
    new Zapador(),
    new Artillero()
};

for (Soldado s : soldados) {
    s.saludar();
}
4. Si sobreescribo un método, ¿puedo invocar el método base para trabajar a partir de su resultado? ... ¿qué palabra clave del lenguaje has usado para invocar al método de la clase base?
🔹 Sí ✔

👉 Se puede usar método del padre + añadir comportamiento

🔹 super

👉 Llama al método de la clase base

class Zapador extends Soldado {

        super.saludar();

5. Al sobreescribir un método en Java, ¿qué restricciones existen sobre los tipos de los parámetros y el tipo de retorno? ¿Qué diferencia hay entre sobreescritura (overriding) y sobrecarga (overloading)? ¿Para qué sirve la anotación @Override y por qué es recomendable usarla siempre?
🔹 Reglas overriding

✔ mismos parámetros
✔ mismo nombre
✔ retorno igual o compatible

🔹 Overriding vs Overloading
Overriding → misma firma, subclase
Overloading → mismo nombre, distintos parámetros
🔹 @Override

👉 Indica sobreescritura
👉 Detecta errores del compilador ✔

💡 Idea:
👉 Overriding = polimorfismo
👉 Overloading = comodidad

11. ¿Qué es la "herencia de interfaces" en Java? ¿Existe "herencia múltiple de interfaces"? Pon un ejemplo de una interfaz Fichero...
🔹 Herencia de interfaces

👉 Una interfaz extiende otra

🔹 Herencia múltiple

✔ Sí en interfaces
❌ No en clases

interface Fichero {
    String leer();
}

interface FicheroEscribible extends Fichero {
    void escribir(String c);
    void eliminar();
}

💡 Idea final:
👉 Interfaces = polimorfismo máximo + flexibilidad
